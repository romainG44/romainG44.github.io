Présentation de HTML et début de la structuration

Html ou HyperText Markup Language est un format de données qui permet de créer des sites web. Depuis la version 3, il permet une compatibilité plus forte avec CSS qui s'occupe de la partie graphique de la création d'une page web. HTML permet donc de structurer les données. Sa création remonte à 1991 grâce à Tim Berners-Lee. Il existe aujourd'hui 5 version. La dernière HTML 5 est celle que nous utiliserons dans cette présentation.

Au départ de tout code, la version de html est visible entre chevrons, <!doctype html>. Cela indique que le site est écrit dans la version 5 de html. Sous cette inscription, une balise racine est indiquée sous la forme <html> Après ces indications, nous pouvons commencer à indiquer le titre de notre page web qui servira de référence pour trouver le site internet. Ce titre s'indique comme suit: <head>

Pour que votre site puisse comporter les caractères spéciaux, pour nous de la langue française, un léger ajout est nécessaire entre l'indication head et l'indication title. Il s'agit de <meta charset="utf-8" />. Cet ensemble indique au navigateur le jeu d'encodage dans lequel la page doit être retranscrite. L'indication utf(Universal Character Set Transformation Format)-8 indique le nombre de bit encodé, dans ce cas 256. L'interaction entre plusieurs codes permet néanmoins de dépasser ce nombre. <title> Indiquez votre titre </title> Ensuite refermer l'entête. </head>

Cet ensemble head est désigné sous le terme "enfant". Il s'agit donc de l'enfant de la balise racine html. Cette racine possède un enfant, il s'agit de la balise body. Cela se repère dans le code par une indentation par rapport à la balise racine. Comme pour toutes les autres balises, elle possède une ouverture et une fermeture. Ces indications d'ouverture et de fermeture vous seront expliquer dans la partie suivante.

1. Présentation des balises

Les balises sous html se repèrent grâce aux chevrons. Elles sont de deux sortes, soient complètes, soient autofermante. Dans le premier cas des balises d'ouverture, par exemple <balises>, et de fermeture, par exemple </balises>, sont nécessaires. Dans le second cas, la balise comporte avant le dernier chevron la marque de fermeture, /, par exemple <balise/>. A l'intérieur des balises, plusieurs niveaux d'écriture peuvent exister. Dans ce cas, il faut respecter l'indentation afin de rendre plus lisible la lecture du fichier. Ils existent deux balises qui sont interdites d'utilisation et qui proviennent d'un héritage des anciennes versions de HTML. Il s'agit des balises <br> et <he>. La première balise interdite sert à ajouter un solide d'un point de vue sémantique. La deuxième balise interdite ajoute une ligne horizontale.

2. Mise en évidence de certains caractères

Nous allons voir maintenant comment mettre en avant plusieurs caractères en évidence. Pour cela, il existe différentes solutions, soit la mise en italique par l'intermédiaire de <em> </em>, soit la mise en gras grâce à <strong> </strong>, soit en surlignant avec <mark> </mark>.

Il est possible d'afficher aussi les chevrons qui servant à encadrer les balises grâce à & lt; et & gt;. Pour les nécessités de l'affichage un espace est nécessaire, mais il est inexistant quand on le code.

Au même titre que certains caractères, on peut mettre en évidence certains paragraphes par la présence soit de titre avec des formats différents, comme il possible de le voir entre le titre de cette partie, ou de ce sous ensemble, et celui du début. Au maximum, il est toléré d'avoir 6 sous ensembles. Ces derniers se déclarent au sein du code comme suit : <h1> TITRE de la partie </h1>.

3. Présentation des listes

Il existe trois listes dispônibles dans HTML, la liste ordonnée, la liste désordonnée et la liste de définition.

  1. La liste ordonnée Elle permet de lister les informations sous forme de titre numéroté. Pour cela, le code à réaliser est le suivant: <ol> <li> puis refermer vos chevrons </li> Enfin refermer les chevrons d'ouverture de la liste </ol>
  2. La liste désordonnée Elle permet de lister les informations sous forme de listes à puces. Pour cela, le code à réaliser est : <ul> Inscriver les éléments de votre liste comme précedemment, puis fermer la liste </ul>.
  3. La liste de définition Cette liste permet comme un dictionnaire et comme son nom l'indique de donner les définitions d'un mot en y insérant selon votre envie des synonymes. En effet, il est possible d'indiquer plusieurs pour une même définition, et inversément. La forme générale du code est donc: <dl> <dt> votre mot à définir</dt> Ajouter autant de nom à définir que vous souhaitez puis donner la définition commune à ces mots. <dd> defintion du mot <dd> Enfin, il faut fermer la liste de définition <dl>
Nous pouvons prendre comme exemple le mot maison:
maison
Bâtiment servant à loger les personnes.

5. Liens

5.1 Lien vers un document

La création de lien se fait par l'intermédiaire d'une nouvelle balise <a>. Pour créer le lien hypertexte, il nous faut réjouter un petit éléments href. Ce-dernier est suivi du signe = et du lien hypertexte entre guillemets. L'attribut href donne la cible du lien vers lequel le texte qui suit renvoie. Il permet soit d'avoir des hypertextes complets avec des liens possédant des URL complets, soit avec des liens avec des URL partiels (A DEFINIR). Après cette balise, il est nécessaire de rajouter un texte, par exemple un texte qui permet de cliquer vers le lien.

lien vers l'autre le fichier test

lien vers un autre fichier test dans un dossier

5.2 Liens inter document

Ce lien permet d'atterir directement dans la partie souhaitée. Le cas le plus connue est l'utilisation dans le contexte d'un sommaire. Pour faire référence avec la zone souhaitée, il faut d'abord définir l'id de cette partie. Celle-ci se place à l'intérieur d'une balise, par exemple la balise <article>, ou <h1>. Elle se note de la manière suivante : <h1 id="nom_du_lien">. Il est fortement recommandé de ne pas utiliser d'espace dans le nom ainsi que de chiffre comme premier caractère. De même, cette attribut doit être unique. Pour qu'il puisse fonctionner, il faut désormais appeler ce marqueur. Il fonctionne d'une manière similaire au lien. On ouvre la balise <a>. A l'intérieur de cette balise, on utilise l'attribut href. L'ensemble de la balise s'écrit donc de la manière suivante: <article href="# nom_de_l'id" >.

Donnons un exemple. Nous avons un partie sur les cakes salés que l'on souhaite directement appelé depuis le sommaire. Au niveau de cette partie, si nous avons utiliser la balise article, nous aurons alors: <article id="cakes_sales">. Au niveau du sommaire, nous devrions inscire <a href='#cakes_sales'>cakes salés/a>. L'élément avant le cakes_sales est appelé une ancre.

5.3 Liens vers un site avec une ouverture

Pour intégrer des liens qui renvoient à d'autres sites, différents mots clés peuvent être ajoutés pour spécifier l'action que l'on souhaite effectuer en ouvrant ce lien. La commande de base est la même que pour les autres liens, c'est à dire : <a href="URL du lien". La différence est qu'il y a un ajout de commande à la suite de celle-ci.

  • Ouverture de lien dans le même onglet

    Cette action se fait directement. C'est le paramètre par défaut. Si l'on souhaite l'indiquer, il suffit d'ajouter les informations suivantes target = "_self". On ferme ensuite les chevrons.

  • Ouverture dans un nouveau onglet

    Cette action permet de garder sa page ouverte en même temps qu'un nouvel onglet est ouvert. Le code à taper est le suivant: target ="_blank". Comme précédemment, on ferme les chevrons.

  • Remonter vers le haut

    Cette action permet de revenir en haut de la page web. La formule a notée est href="#top" ou target="_top" après la notification href.

  • Remonter vers la navigation parente

    Cette action permet de charger la réponse dans le contexte de navigation parent. En cas d'absence de parent, il se comporte comme le _self. Cela se note au niveau de la manière suivante target="_parent

6. Les images

Pour insérer une image au sein du code, il faut entrer la commande suivante <img src="URL de l'image" alt="texte à afficher">. Les formats pris en charge par cette commande sont, le jpeg, les gifs et ceux animés, PNG, APNG, SVG, BMP, ICO (bitmap), ICO (PNG). Il faut noter que le format BMP est à proscrire car la résolution est sous forme de pixel. La différence entre jpeg et PNG se fait principalement sur la résolution. C'est pour cela, que le jpeg est plus adapté pour la photo. L'avantage du PNG est qu'il gère la transparence, c'est-à-dire que les images peuvent se superposer au texte sans réel soucis. Il est important de garder en mémoire qu'il faut de préférence redimensionner l'image avant de l'intégrer dans le html.

7. les tableaux

Sur HTML, un tableau correspond à un ensemble de cellulle sur plusieurs lignes. Il se définit par la balise <table> et est composé de cellule <td> et de ligne <tr>. Pour créer un tableau de deux lignes et quatres cellules, il faudra donc coder les lignes suivantes:

Tableau d'essai
0 lettre prenom
Fin de première colonne Fin de deuxième colonne
1 A B
2 D E
3 F G H

Les entêtes sont ici symbolisées par la balise <thead> suivi des balises qui encapsulent les annotations du tableau, <th>. Ces entêtes peuvent occuper la première cellule de chaque ligne et serviront dans ce cas à identifier la ligne. De même que l'on peut créer des entêtes, on peut créer des pieds de tableaux. Pour cela il faut utiliser les balises <tfoot>. De manière académique, il convient de mettre l'ensemble des entêtes et des pieds après la balise d'ouverture du tableau.Viennent ensuite les balises de remplissage du tableau. L'attribut rowspan permet d'agrandir la largeur de la cellule et l'attribut colspan la hauteur de la cellule. L'utilisation seule de rowspan va "fusionner" la cellule de la valeur avec la cellule sous-jacente. L'utilisation seule de colspan va "fusionner" la cellule adjacente. L'aggrandissement des cellules a pour effet d'aggrandir les cellules sous-jacentes.

Il est possible de rajouter une légende pour le tableau. La légende se placera directement en début du tableau. Pour cela, l'annotation est la suivante: <caption> Légende </caption>.

Remarque: Il existe la possibilité de créer des groupes de colonnes. L'attribut est colgroup. Ces balises doivent être mises soit après la balise table soit après la balise caption. Ce groupement de colonne se traduit comme cela:

<colgroup>

<col/>

<colspan="2">

</colgroup>

Présentation de CSS et début de la structuration

Le CSS ou Cascading Style Sheet fonctionne avec deux types de boîte principale. Il y a les boîtes en ligne et en bloc. Ces boîtes s'utilisent au sein du fichier html mais ne servent pas à définir un cadre sémantique particulier. La boîte en bloc, ou <div> sert de conteneur. Il faut cependant éviter d'en abuser. La boîte <span> indique un placement des conteneurs en ligne. Par exemple, au sein de site, nous avons utilisons la boîte < div> afin de pouvoir placer des blocs, l'un contenant le titre et les images, et le deuxième, la partie inférieure du site. Ces fonctionnalités au sein de HTML sont restrancrites en CSS grâce à des selecteurs.

1. Les selecteurs

Les selecteurs servent à appeler les éléments qui doivent être impactés par les informations du CSS. Ces selecteurs peuvent être des classes, des id,... Ces sélecteurs permmettent de spécifier des endroits spécifiques sur lesquels le CSS va agir. Dans le cas d'une class il faudra que le nom de la classe soit précéder d'un point ".". Dans le cas d'un id, il faudra un "#". Un ordre de priorité est cependant à respecter. Le selecteur d'un id sera dominant par rapport à un selecteur class. Un selecteur class est dominant sur un selecteur élement. Il est possible de combiner plusieurs selecteurs, via différents opérateurs :

  • le selecteur de voisin immédiat "+".

    Un exemple sera plus parlant. Si nous observons l'expression : dl+dl, cela signifie que toute définition suivie d'une autre définition sera impactée par les instructions du CSS. Si une définition est suivie par autre chose, les instructions ne se réaliseront pas.

  • le selecteur de voisin indirect "~".

    Prenons l'exemple suivant img~a. Le symbol ~ indique le selecteur. Cela indique que les instructions du CSS vont s'appliquer aux enfants "a" de l'img quelque soit sa position au sein de la même balise et du même niveau.

  • Le selecteur de descendant ">".

    Comme son nom l'indique, les effets de l'instruction CSS sont reportés sur le niveau inférieur.Prenons l'exemple de la liste de définition. si en selecteur nous indiquaons dl>div;. Cela indique que le dl est le parent et le div l'enfant, autrement dit tout les div qui sont des descendants directs de dl seront impactés par les instructions du CSS.

  • Le selecteur de descendant à échelons différents " ".

    Prenons un exemple : div p. Nous pouvons remarquer que l'opérateur indiquant la descendance est un espace. Cela implique que tout les paragraphes "p" du parent "div" seront affetés par les instructions du CSS.

  • Le selecteur universel "*".

    Il permet d'appliquer les informations du CSS à l'ensemble du HTML.

2. Pseudoclass

Les pseudoclass permettent de modifier ou de préciser l'état de la class. Grâce au informations rajoutées à la suite de la class, il est possible soit de modifier une couleur en passant la souris dessus, .rouge:hover{};, soit personaliser le lien visité, .a:visited;, soit d'affecter une règle à des enfants particuliers, :nth-child(2n). 3. Les formulaires

Pour créer un formulaire, il faut créer une balise qui permettra d'encadrer les informations à mettre, telles que le nom ou pseudo, le mail, le commentaire, et surtout à ne pas oublier le bouton envoyez.Commençons par créer un cadre pour le nom. Pour cela dans les balises <form>, il faut créer une deuxième balise, nommé input. Cette balise va contenir plusieurs attributs. Ceux que nous utiliserons, sont les attributs type, name et id. Pour indiquer le nom de la case d'un formulaire, il faut utiliser la balise label. Dans la balise <label>, il faut ajouter l'attribut for="" et y intégrer l'id du label.Voici un exemple:

<form action="" method="">

<label for="name">Votre nom</labe>

<input type="text" name="name" id="name"/>

</form>

La procédure est la même si on veut demander à l'utilisateur d'ajouter son adresse mail et un commentaire. Pour l'adresse mail, le type sera "mail". Pour le commentaire, il faut remplacer input par "textarea". Dans le cas du commentaire, il faut fermer la balise. Il ne s'agit pas d'une balise autofermante.

Pour ajouter le bouton envoyez, il suffit d'introduire la balise button, avec le type, pour le cas présent, "submit". Tout comme label, on entre la valeur que l'on souhaite, et on referme ensuite la balise button, </button>.

Les formulaires à choix multiples

Il existe différents types de formulaires à choxs multiples dans les formulaires :

  • Les choix à cocher

    • Dans ce cas, il faut changer le type dans input et le remplacer par "checkbox". A la fin de la balise input, on peut ajouter une valeur avec l'atttibut "value".

  • Les choix à puces

    • La manipulation est la même mais au lieu de remplacer de checkbox, on remplace par "radio".

  • Les choix à menu déroulant

    • Pour ce type de choix, le input ne s'applique, il faut ouvrir une nouvelle balise "select" et y indiquer l'attribut name. Pour avoir des choix, il faut ajouter des balises option. Chaque balise <option> doit contenir une valeur "value". Ensuite on entre la valeur du choix et on referme la balise option.

Pour regrouper ces choix une commande simple est à ajouter. Il faut marquer au début des balises qui démarrent le choix, <optgroup label=""> L'attriut label va permettre de nommer le groupe des choix.

Prenons un exemple, nous voulons un choix déroulant de continent et de pays, pour rester original.

<select name="pays">

<optgroup label="Europe">

<option value="1"> Islande </option>

<option value="2"> Norvège </option>

<option value="3"> Dannemark </option>

<option value="4"> Suède </option>

<option value="5"> Finland </option>

</optgroup>

<optgroup label="Afrique">

<option value="1"> Maroc </option>

<option value="2"> Algérie </option>

<option value="3"> Tunisie </option>

<option value="4"> Lybie </option>

<option value="5"> Egypte </option>

</optgroup>

</select>

Les formulaires à choix simples et imposés

Il est possible de rendre l'ensemble de ses listes de choix multiples en liste simple. Il suffit d'indiquer, dans l'attribut name="", le même nom partout.

Pour imposer un choix dans un formulaire, on ajoute à la fin des balises < input>, <textarea> et <select>.
