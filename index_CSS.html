Présentation de CSS

Présentation de CSS et début de la structuration

Le CSS ou Cascading Style Sheet fonctionne avec deux types de boîte principale. Il y a les boîtes en ligne et en bloc. Ces boîtes s'utilisent au sein du fichier html mais ne servent pas à définir un cadre sémantique particulier. La boîte en bloc, ou <div> sert de conteneur. Il faut cependant éviter d'en abuser. La boîte <span> indique un placement des conteneurs en ligne. Par exemple, au sein de site, nous avons utilisons la boîte < div> afin de pouvoir placer des blocs, l'un contenant le titre et les images, et le deuxième, la partie inférieure du site. Ces fonctionnalités au sein de HTML sont restrancrites en CSS grâce à des selecteurs.

1. Les selecteurs

Les selecteurs servent à appeler les éléments qui doivent être impactés par les informations du CSS. Ces selecteurs peuvent être des classes, des id,... Ces sélecteurs permmettent de spécifier des endroits spécifiques sur lesquels le CSS va agir. Dans le cas d'une class il faudra que le nom de la classe soit précéder d'un point ".". Dans le cas d'un id, il faudra un "#". Un ordre de priorité est cependant à respecter. Le selecteur d'un id sera dominant par rapport à un selecteur class. Un selecteur class est dominant sur un selecteur élement. Il est possible de combiner plusieurs selecteurs, via différents opérateurs :

  • le selecteur de voisin immédiat "+".

    Un exemple sera plus parlant. Si nous observons l'expression : dl+dl, cela signifie que toute définition suivie d'une autre définition sera impactée par les instructions du CSS. Si une définition est suivie par autre chose, les instructions ne se réaliseront pas.

  • le selecteur de voisin indirect "~".

    Prenons l'exemple suivant img~a. Le symbol ~ indique le selecteur. Cela indique que les instructions du CSS vont s'appliquer aux enfants "a" de l'img quelque soit sa position au sein de la même balise et du même niveau.

  • Le selecteur de descendant ">".

    Comme son nom l'indique, les effets de l'instruction CSS sont reportés sur le niveau inférieur.Prenons l'exemple de la liste de définition. si en selecteur nous indiquaons dl>div;. Cela indique que le dl est le parent et le div l'enfant, autrement dit tout les div qui sont des descendants directs de dl seront impactés par les instructions du CSS.

  • Le selecteur de descendant à échelons différents " ".

    Prenons un exemple : div p. Nous pouvons remarquer que l'opérateur indiquant la descendance est un espace. Cela implique que tout les paragraphes "p" du parent "div" seront affetés par les instructions du CSS.

  • Le selecteur universel "*".

    Il permet d'appliquer les informations du CSS à l'ensemble du HTML.

2. Pseudoclass

Les pseudoclass permettent de modifier ou de préciser l'état de la class. Grâce au informations rajoutées à la suite de la class, il est possible soit de modifier une couleur en passant la souris dessus, .rouge:hover{};, soit personaliser le lien visité, .a:visited;, soit d'affecter une règle à des enfants particuliers, :nth-child(2n). 3. Les formulaires

Pour créer un formulaire, il faut créer une balise qui permettra d'encadrer les informations à mettre, telles que le nom ou pseudo, le mail, le commentaire, et surtout à ne pas oublier le bouton envoyez.Commençons par créer un cadre pour le nom. Pour cela dans les balises <form>, il faut créer une deuxième balise, nommé input. Cette balise va contenir plusieurs attributs. Ceux que nous utiliserons, sont les attributs type, name et id. Pour indiquer le nom de la case d'un formulaire, il faut utiliser la balise label. Dans la balise <label>, il faut ajouter l'attribut for="" et y intégrer l'id du label.Voici un exemple:

<form action="" method="">

<label for="name">Votre nom</labe>

<input type="text" name="name" id="name"/>

</form>

La procédure est la même si on veut demander à l'utilisateur d'ajouter son adresse mail et un commentaire. Pour l'adresse mail, le type sera "mail". Pour le commentaire, il faut remplacer input par "textarea". Dans le cas du commentaire, il faut fermer la balise. Il ne s'agit pas d'une balise autofermante.

Pour ajouter le bouton envoyez, il suffit d'introduire la balise button, avec le type, pour le cas présent, "submit". Tout comme label, on entre la valeur que l'on souhaite, et on referme ensuite la balise button, </button>.

Les formulaires à choix multiples

Il existe différents types de formulaires à choxs multiples dans les formulaires :

  • Les choix à cocher

    • Dans ce cas, il faut changer le type dans input et le remplacer par "checkbox". A la fin de la balise input, on peut ajouter une valeur avec l'atttibut "value".

  • Les choix à puces

    • La manipulation est la même mais au lieu de remplacer de checkbox, on remplace par "radio".

  • Les choix à menu déroulant

    • Pour ce type de choix, le input ne s'applique, il faut ouvrir une nouvelle balise "select" et y indiquer l'attribut name. Pour avoir des choix, il faut ajouter des balises option. Chaque balise <option> doit contenir une valeur "value". Ensuite on entre la valeur du choix et on referme la balise option.

Pour regrouper ces choix une commande simple est à ajouter. Il faut marquer au début des balises qui démarrent le choix, <optgroup label=""> L'attriut label va permettre de nommer le groupe des choix.

Prenons un exemple, nous voulons un choix déroulant de continent et de pays, pour rester original.

<select name="pays">

<optgroup label="Europe">

<option value="1"> Islande </option>

<option value="2"> Norvège </option>

<option value="3"> Dannemark </option>

<option value="4"> Suède </option>

<option value="5"> Finland </option>

</optgroup>

<optgroup label="Afrique">

<option value="1"> Maroc </option>

<option value="2"> Algérie </option>

<option value="3"> Tunisie </option>

<option value="4"> Lybie </option>

<option value="5"> Egypte </option>

</optgroup>

</select>

Il est possible de rendre l'ensemble de ses listes de choix multiples en liste simple. Il suffit d'indiquer, dans l'attribut name="", le même nom partout.

Pour imposer un choix dans un formulaire, on ajoute à la fin des balises < input>, <textarea> et <select>.

4. La flexbox

Le système de fonctionnement correspond à des emplilements de blocs. Chaque bloc va posséder des instructions propres à lui-même et au bloc enfant qu'il possède. Cependant les enfants vont posséder leur propre instruction en plus de ceux de leur parents. Pour repérer qu'un fichier fonctionne sous forme de flex, il suffit de regarder l'attribut display. Dans le cas d'un fonctionnement par flex, celui-ci se présentera comme suit: display=flex;. Dans le cas en flex les valeurs de l'attributs ne sont plus à déterminer à l'intérieur des selecteurs. En effet, le flex place les blocs verticalement de manière automatique. Pour leur indiquer un placement à la ligne, ou en colonne, ou avec des espaces entre les blocs, il faut modifier les paramètres dans le conteneur principale via les propriétés flex-direction, flex-wrap, flex-flow et justify-content. Ces propriétés permettent de modifier respectivement :

  • la direction de placement des blocs (de gauche à droite ou l'inverse, de bas en haut ou l'inverse),
  • les retours à la ligne ou leur absence
  • l'axe principale de mise en place des blocs
  • de mettre en page les blocs les uns par rapport aux autres, soient avec des espaces entre eux, soient tous à gauche, ou en haut,..

On peut également les éléments à l'intérieur des conteneurs. Pour cela, il existe l'attribut order, qui est définit comme 1 par défaut, et qui indique la priorité des blocs. Cet attribut existe également pour les conteneurs conteneurs les blocs plus petits. Les attributs flex-grow, flex-shrink, flex-basis, flex et align-self eident à organiser les blocs des conteneurs de manières à la fois indépendantes les uns des autres mais également les uns par rapport aux autres (flex_basis). 5. Les transitions

D'après le site developer.mozilla,

Les transitions CSS permettent de contrôler la vitesse d'animation lorsque les propriétés CSS sont modifiées. Plutôt que le changement soit immédiat, on peut l'étaler sur une certaine période. Ainsi, si on souhaite passer un élément de blanc à noir, on pourra utiliser les transitions CSS afin que cette modification soit effectuée progressivement, selon une courbe d'accélération donnée.

Sur CSS il est possible d'appliquer sur des images simple ou des gif des effets. Pour créer un carré bleu qui se tranforme en carré rose, nous utilisons un pseudo-elements qui va indiquer les effets finaux de la transition. Pour appliquer on utilise dans le selecteur, associé à l'image, auquel on voit appliquer la transition, l'outil transition. Cet outil s'inscrit au sein du selecteur qui doit être affecté par la transformation. Cette dernière doit être définie dans les pseudo-elements.

6. Les animations

Les animations permettent de mettre en mouvement les images ou les objets que l'on souhaitent affecter. Pour cela, il faut utiliser différentes propriétés qui permettent de créer des animations propres aux souhaits des personnes. Il existe par exemple, les propriétes suivantes, Animation.startTime, Animation.currentTime, Animation.effect. Ces propriétés agissent sur le temps d'animation ou le délai avant l'animation, mais aussi sur les effets souhaités. Pour utiliser ces animations, il faut y appliquer des méthodes, telles que la mise en pause de l'animation (Animation.pause()), l'inversion du jeu de l'animation (Animation.reverse()).
